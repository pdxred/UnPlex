<task type="auto">
  <name>Task 2: Fix API task collision with create-per-request pattern</name>
  <files>
    SimPlex/components/screens/HomeScreen.brs

    SimPlex/components/screens/EpisodeScreen.brs

    SimPlex/components/widgets/VideoPlayer.brs
  </files>
  <action>
For each file that currently creates a single PlexApiTask in init() and reuses it (m.apiTask pattern):

1. Remove the `m.apiTask = CreateObject("roSGNode", "PlexApiTask")` line from init() (or equivalent).

2. In each function that makes an API request, create a FRESH task instance:
   ```brightscript
   task = CreateObject("roSGNode", "PlexApiTask")
   task.endpoint = endpoint
   task.params = params
   task.observeField("status", "onCallbackName")
   task.control = "run"
   m.currentApiTask = task  ' Store reference to prevent GC during request
   ```

3. In the observer callback, read response from the event source node:
   ```brightscript
   sub onCallbackName(event as Object)
       task = event.getRoSGNode()
       status = event.getData()
       if status = "completed"
           response = task.response
           ' ... process response
       end if
   end sub
   ```
   Note: If getRoSGNode() does not return the task node (per research open question #3), fall back to reading from m.currentApiTask. Test both approaches -- the stored reference pattern is always safe.

4. If a screen makes multiple DIFFERENT API calls (e.g., HomeScreen loads libraries AND grid content), use different m.* variable names for each (e.g., m.libraryTask, m.gridTask) to allow true concurrency.

5. Files to update and their current m.apiTask usage:
   - HomeScreen.brs: Library listing + grid content loading (most complex -- may have multiple request types)
   - DetailScreen.brs: Metadata fetch
   - EpisodeScreen.brs: Season/episode listing
   - SettingsScreen.brs: Any API calls
   - Sidebar.brs: Library sections fetch
   - VideoPlayer.brs: Media info fetch (may use a different task type -- only change if it uses PlexApiTask)

6. Do NOT change PlexSearchTask, PlexAuthTask, PlexSessionTask, ServerConnectionTask, or ImageCacheTask usage -- those are separate task types with different patterns.

After changes, grep for the old pattern to confirm no single-instance reuse remains.
  </action>

Run: npx bsc

  </verify>
