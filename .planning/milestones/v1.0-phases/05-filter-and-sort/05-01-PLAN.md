Replace the existing FilterBar (ButtonGroup with All/Unwatched) with a persistent text summary bar.

**FilterBar.xml changes:**
- Remove the ButtonGroup child entirely
- Keep the Rectangle background (width 1620, height 50, color 0x232342FF)
- Add a Label id="summaryLabel" inside the rectangle: translation [15, 12], font MediumSystemFont, color 0xA0A0B0FF, width 1400
- Add a Label id="countLabel" right-aligned: translation [1500, 12], font MediumSystemFont, color 0x808090FF, horizAlign right, width 120
- Interface fields to KEEP: sectionId (onChange), filterChanged (alwaysNotify), activeFilters
- Interface fields to ADD:
  - filterState (type assocarray, onChange="onFilterStateChanged") -- the central state object set by bottom sheet
  - totalItems (type integer, onChange="onTotalItemsChanged") -- set by HomeScreen after API response
  - totalFiltered (type integer, onChange="onTotalItemsChanged") -- items matching current filters

**FilterBar.brs changes:**
- Replace entire contents. New init():
  - Find summaryLabel, countLabel nodes
  - Initialize m.filterState = { sort: "titleSort:asc", genre: "", year: "", unwatched: "" }
  - Set m.top.activeFilters = {} and m.top.filterState = m.filterState
  - Call updateSummary()

- onSectionIdChange: Reset m.filterState to defaults, update m.top.filterState, m.top.activeFilters = {}, call updateSummary(), fire m.top.filterChanged

- onFilterStateChanged(event): Read m.top.filterState into m.filterState. Call buildFilterParams() to produce API params AA. Set m.top.activeFilters = params. Call updateSummary(). Fire m.top.filterChanged = m.top.activeFilters.

- buildFilterParams() as Object: Build params AA from m.filterState:
  - Always include "sort": m.filterState.sort
  - If m.filterState.genre <> "" then params["genre"] = m.filterState.genre
  - If m.filterState.year <> "" then params["year"] = m.filterState.year
  - If m.filterState.unwatched <> "" then params["unwatched"] = m.filterState.unwatched
  - Return params

- buildFilterSummary() as String: Build display text per user decision format:
  - If genre set: add "Genre: " + genre display names (store m.genreDisplayNames as AA mapping keys to names, set by bottom sheet via a new interface field genreNames type assocarray)
  - If unwatched = "1": add "Unwatched"
  - If year set: add "Year: " + year display value
  - If no filters active: text = "All"
  - Join parts with " . " (dot-separated per user spec)
  - Append " . Sort: " + getSortDisplayName(m.filterState.sort)
  - Return full string

- getSortDisplayName(sortValue as String) as String: Map sort values to display:
  - "titleSort:asc" -> "Title A-Z"
  - "titleSort:desc" -> "Title Z-A"
  - "addedAt:desc" -> "Date Added (Newest)"
  - "addedAt:asc" -> "Date Added (Oldest)"
  - "year:desc" -> "Year (Newest)"
  - "year:asc" -> "Year (Oldest)"
  - "rating:desc" -> "Rating (Highest)"
  - "rating:asc" -> "Rating (Lowest)"

- updateSummary(): Set m.summaryLabel.text = buildFilterSummary(). Update count label.

- onTotalItemsChanged(): Update m.countLabel.text. If m.top.totalFiltered < m.top.totalItems then show "X of Y" format, else show "X items".

- Add interface field genreNames (type assocarray) for the bottom sheet to pass genre key->name mappings.

- Keep onKeyEvent returning false (FilterBar is not focusable, just a display bar).
