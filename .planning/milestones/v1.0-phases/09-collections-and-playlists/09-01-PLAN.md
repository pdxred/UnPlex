  <name>Task 1: Add collections view mode to HomeScreen</name>
  <files>SimPlex/components/screens/HomeScreen.brs</files>
  <action>
**Add collection browsing as a view mode in HomeScreen. Collections are accessed within a library view.**

1. **Add collection state variables in init():**
   - `m.isCollectionsView = false`
   - `m.collectionRatingKey = ""` (when viewing collection contents)

2. **New function loadCollections():** Fetch collections for current library section:
   ```brightscript
   sub loadCollections()
       if m.isLoading then return
       m.isLoading = true
       m.loadingSpinner.visible = true
       m.emptyState.visible = false

       endpoint = "/library/sections/" + m.currentSectionId + "/collections"
       params = {
           "X-Plex-Container-Start": m.currentOffset.ToStr()
           "X-Plex-Container-Size": m.global.constants.PAGE_SIZE.ToStr()
       }

       m.retryContext = { endpoint: endpoint, params: params, handler: "onCollectionsLoaded", requestType: "collections" }

       task = CreateObject("roSGNode", "PlexApiTask")
       task.endpoint = endpoint
       task.params = params
       task.observeField("status", "onCollectionsLoaded")
       task.control = "run"
       m.currentApiTask = task
   end sub
   ```

3. **New function onCollectionsLoaded(event):** Parse collection items:
   - Check `state = "completed"`, hide spinner, reset retry count
   - On error: follow existing retry pattern (auto-retry once, then disconnect or error dialog)
   - Parse `response.MediaContainer.Metadata` array
   - Build ContentNode tree for PosterGrid:
     - `ratingKey`, `title`, `itemType = "collection"`, `thumb` (composite art)
     - `childCount` for subtitle/count display
     - Build poster URL via `BuildPosterUrl(item.thumb, c.POSTER_WIDTH, c.POSTER_HEIGHT)`
   - Set `m.posterGrid.content = content`
   - Show empty state if no collections: "No collections found" / "Create collections in Plex to organize your library"

4. **New function loadCollectionContents(collectionKey):** Fetch items inside a collection:
   ```brightscript
   sub loadCollectionContents(collectionKey as String)
       m.collectionRatingKey = collectionKey
       m.currentOffset = 0
       m.isLoading = true
       m.loadingSpinner.visible = true

       endpoint = "/library/collections/" + collectionKey + "/children"
       params = {
           "X-Plex-Container-Start": "0"
           "X-Plex-Container-Size": m.global.constants.PAGE_SIZE.ToStr()
       }

       m.retryContext = { endpoint: endpoint, params: params, handler: "onApiTaskStateChange", requestType: "library" }

       task = CreateObject("roSGNode", "PlexApiTask")
       task.endpoint = endpoint
       task.params = params
       task.observeField("status", "onApiTaskStateChange")
       task.control = "run"
       m.currentApiTask = task
   end sub
   ```
   - Collection contents use the same `onApiTaskStateChange` -> `processApiResponse` path as library items (same item schema)

5. **Modify onGridItemSelected():** When `m.isCollectionsView = true` and the selected item has `itemType = "collection"`:
   - Call `loadCollectionContents(item.ratingKey)`
   - Set `m.isCollectionsView = false` (now viewing collection contents, not the collection list)
   - This makes back navigation work: pressing back from collection contents returns to collection list

6. **Add collections toggle via filter bar or sidebar:** When viewing a library (libraryOnly mode), add a keyboard shortcut or option to toggle between library items and collections:
   - In `onKeyEvent()`, handle the "options" key when in libraryOnly mode:
     - If NOT in collections view AND `m.isSheetOpen` is false: show options with "Filter" and "Collections" choices
     - Actually, simpler: the `onSpecialAction` handler can accept a "viewCollections" action from the sidebar
   - Better approach per CONTEXT.md: Add a "Collections" action to the filter bar area. Since options key already opens filter bottom sheet, add a dedicated sidebar-level mechanism:
     - When user selects a library from sidebar, if they press OK again on the same library, toggle to collections view for that library
     - OR: Add to the hub section of sidebar alongside "Home": "Collections" item that takes sectionId context

   **Simplest approach per CONTEXT.md decision:** Add a "Collections" toggle to the existing sidebar hub section. When user clicks "Collections" while viewing a library, switch to collections view.

   In `Sidebar.brs setupStaticLists()`, add "Collections" as second hub item:
   ```brightscript
   node = hubContent.createChild("ContentNode")
   node.title = "  Collections"
   ```

   In `onHubSelected()`:
   ```brightscript

       m.top.specialAction = "viewHome"
   else if index = 1
       m.top.specialAction = "viewCollections"
   end if
   ```

   Update `layoutLists()` to account for 2 hub items:
   ```brightscript
   hubHeight = 2 * 50  ' 2 hub items (Home, Collections)
   ```

7. **Handle "viewCollections" in HomeScreen onSpecialAction():**
   ```brightscript
   else if action = "viewCollections"
       if m.currentSectionId <> ""
           m.isCollectionsView = true
           m.currentOffset = 0
           m.viewMode = "libraryOnly"
           onViewModeChanged()
           loadCollections()
       end if
   ```

8. **Modify onKeyEvent() for back navigation in collections:**
   - When in collections view and user presses back: return to library grid (set `m.isCollectionsView = false`, reload library)
   - When viewing collection contents (m.collectionRatingKey <> "") and user presses back: return to collections list
   ```brightscript
   else if key = "back"
       if m.collectionRatingKey <> ""
           ' Return to collections list from collection contents
           m.collectionRatingKey = ""
           m.isCollectionsView = true
           m.currentOffset = 0
           loadCollections()
           return true
       else if m.isCollectionsView
           ' Return to library from collections list
           m.isCollectionsView = false
           m.currentOffset = 0
           loadLibrary()
           return true
       end if
       m.top.navigateBack = true
       return true
   ```

9. **Handle loadMore in collections view:** In onLoadMore(), if `m.isCollectionsView`, return (no pagination for collections view — they're typically small lists). If `m.collectionRatingKey <> ""`, paginate collection contents using existing library pagination.

   - Hide hub rows and filter bar when showing playlists
   - Set `m.isPlaylistsView = true`
   - Set `m.viewMode = "libraryOnly"` (reuse same layout)
