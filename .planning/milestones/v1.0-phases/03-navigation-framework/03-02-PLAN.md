<task type="auto">
  <name>Task 1: Hub item selection, focus management, and auto-refresh</name>
  <files>SimPlex/components/screens/HomeScreen.brs</files>
  <action>
Add hub row item selection handling, focus management between RowList and grid, auto-refresh timer, and refresh on return from playback.

**Hub Item Selection:**
1. In init(), observe hubRowList itemSelected: `m.hubRowList.observeField("itemSelected", "onHubItemSelected")`
2. Add `onHubItemSelected(event as Object)` sub:
   - Get selectedInfo = event.getData() (returns [rowIndex, itemIndex] array)
   - Look up hubType from m.hubRowMap[selectedInfo[0].ToStr()]
   - Get content node: rowContent = m.hubRowList.content.getChild(selectedInfo[0]), itemContent = rowContent.getChild(selectedInfo[1])
   - If hubType = "continueWatching": set m.top.itemSelected = { action: "play", ratingKey: itemContent.ratingKey, itemType: itemContent.itemType, viewOffset: itemContent.viewOffset } (per locked decision: resume playback immediately)
   - Else (onDeck or recentlyAdded): set m.top.itemSelected = { action: "detail", ratingKey: itemContent.ratingKey, itemType: itemContent.itemType } (per locked decision: open detail screen)

**Focus Management (onKeyEvent updates):**
Rewrite onKeyEvent to handle three focus areas (sidebar, hubs, grid):

- `key = "down"`:
  - If focusArea = "hubs": check if on last hub row via m.hubRowList.rowItemFocused[0] >= m.hubRowCount - 1. If so, set focusArea = "grid", m.posterGrid.setFocus(true), return true.
  - (Let RowList handle down within hub rows naturally)

- `key = "up"`:
  - If focusArea = "grid": check if grid is near top (m.posterGrid has itemFocused — check if it's in the first row, i.e., itemFocused < GRID_COLUMNS from constants). If so and m.hubRowCount > 0, set focusArea = "hubs", m.hubRowList.setFocus(true), return true. If m.hubRowCount = 0, don't transfer (no hub rows to go to).

- `key = "left"`:
  - If focusArea = "hubs": check m.hubRowList.rowItemFocused[1] = 0 (first item in row). If so, set focusArea = "sidebar", m.sidebar.setFocus(true), return true. (Per locked decision: left on first hub item opens sidebar)
  - If focusArea = "grid": set focusArea = "sidebar", m.sidebar.setFocus(true), return true

- `key = "right"`:
  - If focusArea = "sidebar":
    - If m.hubRowCount > 0 and m.viewMode = "hubGrid": set focusArea = "hubs", m.hubRowList.setFocus(true), return true
    - Else: set focusArea = "grid", m.posterGrid.setFocus(true), return true

- `key = "back"`: unchanged (m.top.navigateBack = true)

**Auto-Refresh Timer:**
1. In init(), create Timer node for periodic hub refresh:
   ```
   m.hubRefreshTimer = CreateObject("roSGNode", "Timer")
   m.hubRefreshTimer.duration = 120  ' 2 minutes per research recommendation
   m.hubRefreshTimer.repeat = true
   m.hubRefreshTimer.observeField("fire", "onHubRefreshTimer")
   m.hubRefreshTimer.control = "start"
   ```
2. Add `onHubRefreshTimer(event as Object)` sub: if not m.isLoadingHubs, call loadHubs()

**Refresh on Return from Playback:**
Update onFocusChange to detect when HomeScreen regains focus (e.g., after returning from playback or detail screen) and reload hubs:
- When m.top.isInFocusChain() and m.top.focusedChild = invalid:
  - Call loadHubs() to refresh hub data (uses existing isLoadingHubs guard)
  - Then delegate focus to the appropriate area based on m.focusArea

**Scroll Position Preservation (per locked decision):**
Before calling loadHubs() for refresh (not initial load), save the current rowItemFocused value:
- `m.savedHubFocus = m.hubRowList.rowItemFocused`
After buildHubRowContent() completes, if m.savedHubFocus is valid and row count hasn't changed, restore via:
- `m.hubRowList.jumpToRowItem = m.savedHubFocus`
Reset m.savedHubFocus to invalid after restore.

**View Mode State:**
1. In init(), add `m.viewMode = "hubGrid"` (default per locked decision: hub rows at top, library grid below)

**Cleanup Updates:**
In cleanup(), add:
- Stop and unobserve hubRefreshTimer
- Unobserve hubRowList itemSelected
- Stop and unobserve hubsTask if not invalid

**Access to PosterGrid Internal State:**
To check if the grid is at the top row, access m.posterGrid's internal grid. Add a helper or use the PosterGrid's content and focused index. The PosterGrid widget wraps a MarkupGrid and exposes itemSelected/loadMore. For focus detection, read `m.posterGrid.findNode("grid").itemFocused` — if it's < GRID_COLUMNS, user is on the first row.

Note: Use `m.global.constants.GRID_COLUMNS` for the column count (currently 6).
