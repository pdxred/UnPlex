**VideoPlayer.brs - scrobble()**: After `task.control = "run"`, add watchStateUpdate emission to broadcast to all screen observers:
```brightscript
' Broadcast watch state update to all visible screens
m.global.watchStateUpdate = {
    ratingKey: m.top.ratingKey
    viewCount: 1
    viewOffset: 0
}
```

Also add watchStateUpdate emission in signalPlaybackComplete() (created in Plan 01). Before setting playbackResult, emit the current progress for stopped/cancelled cases:
```brightscript
' Emit final watch state before signaling completion
if reason = "finished"
    ' Already handled by scrobble() which fires before signalPlaybackComplete
else if reason = "stopped" or reason = "cancelled"
    m.global.watchStateUpdate = {
        ratingKey: m.top.ratingKey
        viewCount: 0
        viewOffset: m.currentPosition
    }
end if
```
This way, when the user stops a video partway through, the progress bar on the poster updates immediately without waiting for a page refresh. Do NOT emit on every reportProgress() call (every 10 seconds) as that would cause excessive re-renders of all grid items.
