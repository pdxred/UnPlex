**VideoPlayer.brs changes:**

1. **In init():** Add references to skip button nodes. Initialize marker storage:
   - `m.skipButton = m.top.findNode("skipButton")`
   - `m.skipButtonLabel = m.top.findNode("skipButtonLabel")`
   - `m.skipButtonBg = m.top.findNode("skipButtonBg")`
   - `m.skipButtonFocus = m.top.findNode("skipButtonFocus")`
   - `m.skipFadeIn = m.top.findNode("skipFadeIn")`
   - `m.skipFadeOut = m.top.findNode("skipFadeOut")`
   - `m.skipFadeOut.observeField("state", "onSkipFadeOutComplete")`
   - `m.introMarker = invalid` (AA with startMs, endMs)
   - `m.creditsMarker = invalid` (AA with startMs, endMs)
   - `m.skipButtonVisible = false`
   - `m.skipButtonType = ""` ("intro" or "credits")
   - `m.skipButtonFocused = false`

2. **New function fetchMarkers():** Called from processMediaInfo() after setting up playback (before m.video.control = "play"). Creates a PlexApiTask to GET `/library/metadata/{ratingKey}/markers`. Observes status field for completion callback.

   ```brightscript
   sub fetchMarkers()
       task = CreateObject("roSGNode", "PlexApiTask")
       task.endpoint = "/library/metadata/" + m.top.ratingKey + "/markers"
       task.observeField("status", "onMarkersLoaded")
       task.control = "run"
       m.markersTask = task
   end sub
   ```

3. **New function onMarkersLoaded(event):** Parse the response.MediaContainer.Marker array. For each marker, check `type` field:
   - If "intro": store `{ startMs: marker.startTimeOffset, endMs: marker.endTimeOffset }` in m.introMarker
   - If "credits": store same pattern in m.creditsMarker
   - Handle missing/empty Marker array gracefully (just return, no error)

4. **Extend onPositionChange():** After updating m.currentPosition, call `checkMarkers()`.

5. **New function checkMarkers():** Compare m.currentPosition against intro and credits marker ranges:
   - If position is within intro marker range AND skip button not already showing for intro: call `showSkipButton("intro")`
   - If position is within credits marker range AND skip button not already showing for credits: call `showSkipButton("credits")`
   - If position is outside ALL marker ranges AND skip button is showing: call `hideSkipButton()`
   - Skip check entirely if m.isTranscodePivotInProgress is true (don't show during PGS pivot)
   - Skip check if TrackSelectionPanel is NOT visible but track panel IS visible, still show button but don't focus it

6. **New function showSkipButton(markerType as String):**
   - Set label text: "Skip Intro" for "intro", "Skip Credits" for "credits"
   - Set m.skipButtonType = markerType
   - Set m.skipButton.visible = true
   - Run m.skipFadeIn.control = "start" for fade-in
   - Set m.skipButtonVisible = true
   - If TrackSelectionPanel is NOT visible: set focus to skip button (`m.skipButton.setFocus(true)`), set m.skipButtonFocused = true, show focus ring
   - If TrackSelectionPanel IS visible: just show button without stealing focus, m.skipButtonFocused = false

7. **New function hideSkipButton():**
   - If not visible, return
   - Run m.skipFadeOut.control = "start" for fade-out
   - m.skipButtonVisible = false
   - m.skipButtonFocused = false
   - Return focus to video node if skip button had focus

8. **New function onSkipFadeOutComplete(event):** When fade-out animation completes (state = "stopped"), set m.skipButton.visible = false and m.skipButton.opacity = 0.0

9. **New function handleSkipPress():** Called when OK pressed while skip button focused:
   - Get the end position from the active marker (m.introMarker.endMs or m.creditsMarker.endMs based on m.skipButtonType)
   - Seek to that position: `m.video.seek = endMs / 1000`
   - Call hideSkipButton()

10. **Extend onKeyEvent():** BEFORE the track panel check, add skip button handling:
    - If m.skipButtonFocused and key = "OK": call handleSkipPress(), return true
    - If m.skipButtonFocused and key = "back": call hideSkipButton(), return true (dismiss without seeking)
    - If m.skipButtonVisible and NOT m.skipButtonFocused and key = "OK": focus the skip button (user pressed OK while button visible but unfocused)

11. **Extend processMediaInfo():** Add `fetchMarkers()` call after the media info is processed, before starting playback. This ensures markers are fetched in parallel with playback start (fire-and-forget pattern, markers arrive during early playback).

12. **Handle stopPlayback():** Reset marker state — set m.introMarker = invalid, m.creditsMarker = invalid, hideSkipButton if visible.

**Important implementation notes:**
- The skip button focus ring should use the accent color from constants (FOCUS_RING/ACCENT = 0xE5A00DFF)
- checkMarkers() is called on every position change (~250ms) so keep it lightweight — just integer comparisons
- Don't show skip button if m.introMarker/m.creditsMarker is invalid
- When both markers exist, only one button shows at a time based on current position
