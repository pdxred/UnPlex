**PlexApiTask.brs — Add PUT support:**
In the `run()` sub, the existing code handles "POST" and "GET" methods. Add "PUT" handling following the same pattern as PlexSessionTask.brs:
- When method = "PUT", use `url.AddHeader("X-HTTP-Method-Override", "PUT")` then `url.AsyncPostFromString("")`
- This is the established pattern in the codebase (see PlexSessionTask.brs line 34)

**VideoPlayer.brs — PGS transcode pivot:**

1. **State tracking for transcode pivot:**
   Add to init():
   - `m.isTranscodePivotInProgress = false` — prevents rapid PGS switching race conditions
   - `m.previousPlaybackState = invalid` — stores { url, streamFormat, position, audioId, subtitleId } for revert on failure
   - `m.isTranscoding = false` — tracks current playback mode (direct vs transcode)

2. **buildTranscodeUrlWithSubtitles(subtitleStreamID, offsetMs):**
   New function (do NOT modify existing buildTranscodeUrl). Builds URL like:
   ```
   {serverUri}/video/:/transcode/universal/start.m3u8
     ?path={mediaKey}&mediaIndex=0&partIndex=0
     &protocol=hls&directPlay=0&directStream=1
     &videoQuality=100&maxVideoBitrate=20000&videoResolution=1920x1080
     &subtitleStreamID={subtitleStreamID}&subtitles=burn
     &offset={offsetMs / 1000}
     &X-Plex-Token={token}
   ```

3. **Handle PGS selection in onTrackChanged():**
   When type = "subtitle" AND isBitmap = true:
   - If m.isTranscodePivotInProgress, return (ignore rapid clicks)
   - Set m.isTranscodePivotInProgress = true
   - Store current state in m.previousPlaybackState: { url: current content URL, streamFormat: current format, position: m.currentPosition, audioId: m.selectedAudioId, subtitleId: m.selectedSubtitleId }
   - Record current position: offsetMs = m.currentPosition (already in ms from onPositionChange)
   - Stop current video: m.video.control = "stop"
   - Show loading indicator: Find or create a "Switching subtitles..." label/spinner overlay (use a simple Label + BusySpinner group, positioned center-screen). Set it visible.
   - Build transcode URL: url = buildTranscodeUrlWithSubtitles(streamId, offsetMs)
   - Create new ContentNode with url, streamFormat = "hls", title = m.top.itemTitle
   - Set m.video.content = new content node
   - Start playback: m.video.control = "play"
   - Set m.isTranscoding = true
   - Close the panel: m.trackPanel.visible = false

4. **Handle transcode playback state in onVideoStateChange():**
   When state = "playing" AND m.isTranscodePivotInProgress:
   - Hide the "Switching subtitles..." overlay
   - Set m.isTranscodePivotInProgress = false
   - Update m.selectedSubtitleId and panel's selectedSubtitleId
   - Do NOT re-pause (user expects playback to continue after PGS switch)

   When state = "error" AND m.isTranscodePivotInProgress:
   - Hide the "Switching subtitles..." overlay
   - Show error toast: use StandardMessageDialog with title "Subtitle Unavailable" and message ["Could not load this subtitle track. Reverting to previous settings."] and button ["OK"]
   - Revert: rebuild previous ContentNode from m.previousPlaybackState (url, streamFormat), set m.video.content, seek to stored position, m.video.control = "play"
   - Set m.isTranscodePivotInProgress = false
   - Set m.isTranscoding back to false if reverting to direct play
   - Restore previous selectedSubtitleId

5. **"Switching subtitles..." overlay:**
   Add to VideoPlayer.xml (or create dynamically in brs):
   - A Group (id="transcodingOverlay") centered on screen, initially visible=false
   - Contains: BusySpinner (reuse existing pattern from Phase 4) + Label with text "Switching subtitles..." in TEXT_PRIMARY color
   - Semi-transparent background Rectangle behind it for readability

6. **Handle switching BACK from PGS to SRT or Off:**
   When currently transcoding (m.isTranscoding = true) and user selects a non-bitmap subtitle or "Off":
   - Need to switch back to direct play
   - Same pivot pattern: stop video, record position, rebuild direct play URL (from m.previousPlaybackState or re-derive from cached media info), start playback at recorded position
   - If selecting SRT, also set sidecar subtitle on the new ContentNode
   - Set m.isTranscoding = false

<task type="auto">
  <name>Task 2: Add track persistence via Plex API and forced subtitle auto-enable</name>
  <files>SimPlex/components/widgets/VideoPlayer.brs</files>
  <action>
**Track persistence:**

1. **persistTrackSelection(audioStreamId, subtitleStreamId):**
   New sub in VideoPlayer.brs. After any track change (audio or subtitle) succeeds:
   - Create a new PlexApiTask instance
   - Set endpoint = "/library/parts/" + m.partId.ToStr()
   - Set method = "PUT"
   - Set params = {}
   - If audioStreamId > 0: params["audioStreamID"] = audioStreamId.ToStr()
   - If subtitleStreamId > 0: params["subtitleStreamID"] = subtitleStreamId.ToStr()
   - If subtitleStreamId = 0: params["subtitleStreamID"] = "0" (explicitly set Off)
   - task.control = "run" (fire and forget — no need to observe response, per established pattern from scrobble API)

2. **Call persistTrackSelection after successful track changes:**
   - In onTrackChanged() for audio: after switching audio track, call persistTrackSelection(newAudioId, m.selectedSubtitleId)
   - In onTrackChanged() for subtitle (SRT, successful): call persistTrackSelection(m.selectedAudioId, newSubtitleId)
   - In onVideoStateChange() for PGS (when pivot completes successfully): call persistTrackSelection(m.selectedAudioId, newSubtitleId)
   - Do NOT persist on failure/revert

**Forced subtitle auto-enable:**

3. **checkForcedSubtitles():**
   New sub called at end of processMediaInfo(), after streams are parsed and before playback starts.
   - Get selected audio track's languageTag from m.audioStreams (find the one with selected=true)
   - Get device locale: `locale = CreateObject("roDeviceInfo").GetCurrentLocale()` (returns e.g. "en_US")
   - Extract language prefix: `deviceLang = Left(locale, 2)` (e.g. "en")
   - If audio languageTag starts with deviceLang → no forced sub needed, return
   - Otherwise, search m.subtitleStreams for a track where:
     - `forced = true`
     - `language` tag starts with deviceLang (case-insensitive comparison)
   - If found:
     - Set m.selectedSubtitleId to that track's ID
     - If it's text (not bitmap): apply sidecar subtitle (same logic as SRT selection in Plan 01)
     - If it's bitmap: trigger PGS transcode pivot (same logic as PGS handling)
     - Update panel's selectedSubtitleId
   - If no forced track found: do nothing (respect server's default selection)

4. **Handle missing language metadata:**
   In parseStreams(), if a stream has no `language` or `languageTag`:
   - Set language = "Unknown"
   - Set languageTag = ""
   - For displayTitle fallback: if displayTitle is empty, construct as "Track {index+1} ({codec})" for audio or "Track {index+1} ({codec})" for subtitle
   - This ensures the panel always shows meaningful labels
