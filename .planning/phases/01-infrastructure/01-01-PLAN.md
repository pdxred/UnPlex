<task type="auto">
  <name>Task 2: Configure BrighterScript compiler and VSCode F5 deploy</name>
  <files>bsconfig.json, .vscode/launch.json, .env</files>
  <action>
Create bsconfig.json at repo root with:
- $schema pointing to BrighterScript schema URL
- rootDir: "SimPlex" (the subdirectory containing the Roku app)
- stagingDir: "out/staging"
- retainStagingDir: false
- sourceMap: true
- files array: ["manifest", "source/**/*.brs", "components/**/*.brs", "components/**/*.xml", "images/**/*"]
- diagnosticFilters: initially empty codes array for "**/*.brs" src pattern (will be populated after first compilation)
- Do NOT include a deploy section in bsconfig.json -- deploy credentials go in launch.json via .env

Run `npx bsc` to compile. If diagnostic errors appear on existing code, note the error codes and add them to diagnosticFilters.codes array. Iterate until compilation succeeds (zero errors, warnings acceptable). Common codes to expect: 1001 (cannot find name), 1067 (type mismatch), 1061 (property does not exist).

Create .vscode/ directory if it does not exist.

Create .vscode/launch.json with:
- Single configuration of type "brightscript", request "launch"
- name: "SimPlex: Deploy to Roku"
- envFile: "${workspaceFolder}/.env"
- host: "${env:ROKU_HOST}"
- password: "${env:ROKU_PASSWORD}"
- rootDir: "${workspaceFolder}/SimPlex"
- files array matching bsconfig.json files list

Create .env at repo root with placeholder values:
```
# Roku device credentials -- DO NOT commit this file
ROKU_HOST=192.168.1.xxx
ROKU_PASSWORD=your-roku-dev-password
ROKU_USERNAME=rokudev
```
  </action>
  <verify>
Run: npx bsc
Compilation should complete with zero errors (warnings are acceptable). Output staged to out/staging/.
Verify .vscode/launch.json exists and contains "brightscript" type.
Verify .env exists with placeholder values.
  </verify>
  <done>BrighterScript compiles all existing .brs files with zero errors. launch.json configured for F5 deploy with .env credentials. .env template exists with placeholders.</done>
